# 实验步骤

## 1. 实验环境

本实验使用 Python 作为主要开发语言，使用 `uv` 进行项目管理和依赖管理。

推荐环境：

```bash
Python >= 3.10
uv
```

常用 Python 包包括：

```bash
numpy
pandas
scipy
scikit-learn
matplotlib
seaborn
openpyxl
joblib
```

项目初始化示例：

```bash
uv init liquid-identification
cd liquid-identification

uv add numpy pandas scipy scikit-learn matplotlib seaborn openpyxl joblib
```

---

# 2. 实验整体流程

整体实验流程如下：

```text
读取 data/ 数据
    ↓
数据预处理
    ↓
关键频段分析
    ↓
数据分析与降维可视化
    ↓
特征提取
    ↓
训练集 / 验证集 / 测试集划分
    ↓
模型训练
    ↓
模型验证与评估
```

---

# 3. 读取数据

首先读取 `data/` 目录下的实验数据。

数据中应包含：

* 不同液体的频谱数据
* 多组重复实验数据
* 不同条件下的测试数据

读取目标是将原始数据整理成统一格式：


| Sample ID   | Liquid Type | Frequency | Amplitude |
| ----------- | ----------- | --------- | --------- |
| sample\_001 | NaCl        | 30        | 0.52      |
| sample\_001 | NaCl        | 31        | 0.61      |
| ...         | ...         | ...       | ...       |

其中：

```text
Liquid Type
```

作为模型训练时的标签。

---

# 4. 数据预处理

读取数据后，需要进行数据预处理。

主要包括：

---

## 4.1 去噪

去除异常尖峰和明显错误点。

可使用：

* 异常值检测
* 中值滤波
* IQR 方法
* Z-score 异常检测

目的：

```text
减少测量噪声对模型训练的影响
```

---

## 4.2 平滑处理

对频谱曲线进行平滑，降低随机波动。

推荐方法：

* Moving Average
* Savitzky-Golay Filter

其中 Savitzky-Golay 更适合保持峰值和谷值形态。

---

## 4.3 Normalize

为了减少不同实验之间的幅值偏移，需要进行归一化处理。

可选方法：

* Min-Max Normalization
* Z-score Standardization

推荐优先使用：

```text
Z-score
```

因为后续 SVM、PCA 等方法对数据尺度较敏感。

---

# 5. 关键频段分析

根据已有实验结果，重点关注以下频段：

```text
180–220 MHz
270–300 MHz
```

这两个频段中，不同液体表现出明显差异。

分析内容包括：

* 不同液体在关键频段的均值差异
* 峰值位置差异
* 谷值深度差异
* 曲线斜率差异
* 频段能量差异
* 类间距离差异

可使用的方法：

* 方差分析 ANOVA
* Fisher Score
* Mutual Information
* Feature Importance
* 类间距离分析

目标是找出：

```text
哪些频率点或频段最适合区分不同液体
```

---

# 6. 数据分析与可视化

在模型训练前，需要对 `data/` 下的数据进行整体分析。

## 6.1 曲线可视化

绘制不同液体的频谱曲线，包括：

* 原始曲线
* 平滑后曲线
* Normalize 后曲线

用于观察不同液体之间的整体差异。

---

## 6.2 PCA 分析

使用 PCA 对高维频谱数据进行降维。

目的：

* 观察不同液体是否在低维空间中可分
* 判断数据是否具备分类基础
* 查看是否存在异常样本

输出：

* PCA 2D 分布图
* PCA 3D 分布图
* 各主成分解释方差比例

---

## 6.3 t-SNE / UMAP 分析

如果 PCA 分离效果不明显，可以进一步使用：

* t-SNE
* UMAP

用于观察非线性可分性。

---

## 6.4 相关性分析

分析不同频率点之间的相关性，减少冗余特征。

可输出：

* 频率相关性热力图
* 高相关特征筛选结果

---

# 7. 特征提取

完成预处理后，需要将频谱曲线转换为可用于模型训练的特征向量。

---

## 7.1 全频段特征

使用完整 30–300 MHz 的频谱幅值作为输入特征。

即：

```text
X = [A30, A31, A32, ..., A300]
```

优点：

* 保留完整信息
* 适合 SVM、Random Forest、XGBoost

---

## 7.2 关键频段特征

提取重点频段：

```text
180–220 MHz
270–300 MHz
```

作为主要特征。

优点：

* 降低噪声影响
* 提高模型稳定性
* 增强可解释性

---

## 7.3 统计特征

对每个关键频段提取：

* Mean
* Max
* Min
* Variance
* Standard Deviation
* Area
* Energy
* Peak Position
* Valley Position
* Slope

最终形成：

```text
Feature Vector
```

例如：

```text
[mean_180_220, max_180_220, min_180_220, slope_270_300, energy_270_300, ...]
```

---

# 8. 数据集划分

完成特征提取后，将数据整理为：

```text
X: 特征矩阵
y: 液体类别标签
```

然后划分数据集。

建议比例：

```text
训练集：70%
测试集：30%
```

训练集内部再划分验证集：

```text
训练集：80%
验证集：20%
```

最终整体比例大致为：

```text
训练集：56%
验证集：14%
测试集：30%
```

也可以根据数据量调整：

如果数据较少：

```text
训练集：60%
验证集：20%
测试集：20%
```

如果数据较多：

```text
训练集：70%
验证集：15%
测试集：15%
```

划分时需要使用：

```text
stratified split
```

保证每种液体在训练集、验证集、测试集中比例一致。

---

# 9. 模型训练（核心）

模型训练是本实验的核心部分。

目标是建立：

```text
频谱数据 / 特征向量 → 液体名称
```

的分类模型。

---

## 9.1 推荐模型

优先使用：

### SVM

适合小样本、高维频谱分类。

---

### Random Forest

适合特征重要性分析，可解释性较强。

---

### XGBoost

适合后期提高分类性能。

---

### 1D-CNN

数据量足够时，可以直接输入频谱序列进行端到端训练。

---

## 9.2 训练流程

```text
训练数据输入
    ↓
模型学习不同液体的频谱特征
    ↓
验证集调参
    ↓
保存最优模型
```

训练过程中需要使用验证集进行：

* 参数选择
* 特征组合选择
* 模型选择
* 防止过拟合

---

# 10. 模型验证与评估

训练完成后，使用测试集进行最终验证。

测试集不能参与模型训练和调参。

评估指标包括：

* Accuracy
* Precision
* Recall
* F1-score
* Confusion Matrix

重点关注：

```text
模型是否能够正确输出液体名称
```

例如：

```text
Input:
未知频谱数据

Output:
NaCl
```

同时输出：

```text
Prediction Confidence
```

---

# 11. 实验输出结果

最终实验应输出：

* 预处理后的数据
* 关键频段分析结果
* PCA / t-SNE 可视化结果
* 特征向量数据集
* 训练好的分类模型
* 测试集分类报告
* 混淆矩阵
* 未知液体预测结果

最终目标是完成：

```text
输入频谱数据 → 输出对应液体名称
```

的完整模型流程。
